# WhatsApp Latency Improvements Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Reduce WhatsApp AI response latency via three targeted changes: parallelize orchestrator setup queries, send a Meta read receipt immediately on message receipt, and add a `whatsappModel` override so a faster LLM model can be used for WhatsApp without affecting web AI search.

**Architecture:** The Meta webhook path (`processInbound` in `whatsapp.route.ts`) feeds into `runOrchestrator`. Currently `resolveAIConfig`, `resolveChainContext`, and `session.load` run sequentially before the first LLM call — they're independent and can be parallelised. The read receipt is a fire-and-forget Meta API call made as soon as the message is received. The `whatsappModel` override lives on the existing `orgAIConfig` / `systemAIConfig` / `propertyAIConfig` rows and is applied in the orchestrator when `channel === 'whatsapp'`.

**Tech Stack:** Fastify, Prisma (PostgreSQL), Redis, Next.js 14 (admin UI), TypeScript, Meta Cloud API.

```typescript
/**
 * Mark an inbound WhatsApp message as read (shows blue ✓✓ to the sender immediately).
 * Fire-and-forget — failures are logged but never thrown.
 */
export async function markMessageRead(
  phoneNumberId: string,
  accessToken: string,
  messageId: string,
): Promise<void> {
  const url = `https://graph.facebook.com/${META_API_VERSION}/${phoneNumberId}/messages`
  const res = await fetch(url, {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${accessToken}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      messaging_product: 'whatsapp',
      status: 'read',
      message_id: messageId,
    }),
  })
  if (!res.ok) {
    const body = await res.text()
    logger.warn({ phoneNumberId, messageId, status: res.status, body }, '[WhatsApp] markMessageRead failed')
  }
}
```
